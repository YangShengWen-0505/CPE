# 🧮 Concept: Modular Arithmetic & Power Cycles (模運算與次方週期)

> **核心目標**：計算巨大數字的次方和（如 $\sum n^n$）的「最後一位數」。透過數學的週期性，將無限大的運算化簡為常數時間的查表或小範圍迴圈。



---

## 📐 數學原理與公式

### 1. 次方的尾數週期 (Cycle of Exponents)
任何整數 $x$ 經過不斷次方後，其「最後一位數」（即 $\pmod{10}$）會呈現固定週期。
* 根據歐拉定理與觀察，這個週期長度**最大為 4**。
* 例如：$2^1=2 \rightarrow 2^2=4 \rightarrow 2^3=8 \rightarrow 2^4=16 (\text{尾數 } 6) \rightarrow 2^5=32 (\text{尾數回到 } 2)$。
* **核心公式**：求 $B^P \pmod{10}$ 時，可將指數降次為 $P \pmod 4$。
  $$B^P \equiv B^{(P \bmod 4)} \pmod{10}$$
  *(註：若餘數為 0，等同於次方為 4)*

### 2. $n^n$ 的總體週期性 (Overall Periodicity)
若觀察數列 $f(n) = n^n \pmod{10}$，因為：
1. 底數 $n$ 每 10 個一循環（$n \equiv n+10 \pmod{10}$）。
2. 指數 $n$ 每 4 個一循環（次方週期為 4）。
這兩個週期的最小公倍數（LCM）為 $20$。因此 $n^n$ 的尾數每 20 個數字就會一模一樣地重複。

更進一步，每 100 個一組的總和，其最後一位數必定為 0：
$$\sum_{i=1}^{100} i^i \equiv 0 \pmod{10}$$

## 🧠 邏輯推導與實作策略

面對天文數字等級的輸入 $N$（長度可達 100 位數），我們不能使用 `int` 甚至 `long long` 讀取，必須將其視為**字串 (String)**。

### 步驟 1：字串降維 (捨棄週期)
因為每 100 個數字的最後一位數總和都是 $0$，所以無論 $N$ 有多大，我們只需要關心它**除以 100 的餘數**（也就是字串的「最後兩位數」）。
* 若 $N = 123456789$，只需計算 $\sum_{i=1}^{89} i^i$。
* **邏輯**：截取字串最後兩個字元轉為整數 $n$。

### 步驟 2：尾數計算優化
在計算 $1^1 + 2^2 + \dots + n^n$ 時，不用大數乘法：
1. **略過 10 的倍數**：若 $i$ 的尾數是 $0$，則 $i^i$ 尾數必為 $0$，對總和沒影響，直接 `continue`。
2. **指數化簡**：根據前述週期 4 的定理，計算 $i^i$ 時，指數只需跑 `i % 4` 次。

## 🛠️ 關鍵語法與技巧 (Syntax Tips)

### 1. 字串取最後兩位數
利用 ASCII 碼的特性 `'0' ~ '9'` 轉換為數字，無需複雜運算：
```cpp
int n = 0;
if (N.size() >= 2) {
    n += (N[N.size()-2] - '0') * 10; // 十位數
}
n += N[N.size()-1] - '0';            // 個位數
2. 完美的指數週期對應在迴圈中計算次方時，你可能會面臨 i % 4 == 0 的問題（我們需要它跑 4 次而不是 0 次）。最佳數學解法：$$P = (i - 1) \bmod 4 + 1$$這個公式完美將 0, 1, 2, 3, 4, 5... 映射為 4, 1, 2, 3, 4, 1...，避免了寫繁瑣的 if-else。📌 實戰關聯題目 (Related Problems)UVA 10162 - Last Digit解題思路：將大數輸入轉為字串取最後兩位數（$N \pmod{100}$），再利用 $(i-1) \pmod 4 + 1$ 算出每個數字的次方尾數，累加後取 $\pmod{10}$ 即可。
